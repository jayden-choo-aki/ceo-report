Claude Code · Google Workspace · 1Password 도입을 통한 대표이사 의사결정 지원 체계 구축 (안)

TL;DR · 한 줄 요약

현재도 정기적으로 보고되고 있는 CEO 의사결정 자료에, AI(Claude Code)를 활용하여 분석의 범위와 깊이를 확장함으로써 대표이사의 데이터 기반 신속한 의사결정을 지원하고자 합니다. 우선 샘플 데이터로 PoC를 수행하여 유효성이 확인된 경우 단계적으로 확대하며, 외부 컨설턴트와의 안전한 원격 협업을 위해 Google Workspace 법인계정 · 1Password 법인계정 도입을 함께 요청드립니다.

02 · Context

현황 및 추진 동기

  • 현황: 내부 팀에서 ERP 데이터를 주기적으로 정리·가공하여 잘 구조화된 보고서를 대표이사께 전달하고 있음
  • 한계: 가용 인력·시간 제약 상, 보고할 수 있는 정보의 양과 분석 단면(시계열·세그먼트·교차 비교 등)에 자연스러운 한계 존재
  • 기회: 시장 변동성·의사결정 속도가 중요해지는 환경에서, 대표이사가 더 넓은 범위·더 깊은 단면의 데이터를 적시에 검토할 수 있다면 의사결정 품질·속도 모두 향상 가능
  • 수단: AI 도구(특히 Claude Code)의 성숙으로, 동일 인력 구성에서도 분석 범위·결과물 형식을 다양화하는 것이 현실적 선택지로 등장
  • 선결 과제: 외부 컨설턴트의 원격 협업이 전제되므로, 자료·계정 보안 체계의 사전 정립이 필수

03 · Why

추진 목적 (기대 효과)

구분기대 효과
분석 범위·깊이 확장기존 보고서에 시계열 추세·세그먼트 분석·교차 비교 등 새로운 단면 추가
의사결정 속도HTML 링크 형태로 제공 → 대표이사가 PC·모바일 어디서나 즉시 조회
일관성동일 입력 → 동일 가공 → 보고서 포맷 표준화, 시점별 비교 용이
단계적 검증샘플 PoC → 유효성 판단 → 단계적 확대로 투자 리스크 최소화
자산화산출물(코드·문서·계정)이 모두 회사 자산으로 잔존 → 외부 인력 의존도 ↓
보안 정비도구 도입을 계기로 외부 협업용 계정·자격증명 관리 체계를 일괄 정립
※ 본 안의 핵심은 현재 보고 체계의 대체가 아니라, 분석 가능 범위의 확장입니다. 현행 보고는 유지하면서 AI 활용 분석을 별도 트랙으로 시범 운영합니다.

04 · Success

성공 지표

PoC 단계 — Gate 판단 기준

  • 샘플 데이터 기반 신규 분석 보고서 1~2건 시범 작성
  • 대표이사 검토 후 "현행 보고 대비 추가 가치 있음" 판단 시 확대 결정

확대 단계 — 본운영

  • 정기 보고에 신규 분석 항목 편입 (수·범위는 PoC 결과에 따라 협의)
  • 결과물 접근성: 대표이사가 PC/모바일에서 링크 1회로 열람
  • 보안: 외부 컨설턴트 계약 종료 시 1회 작업으로 모든 접근 권한 회수 가능

06 · How it works

작업 흐름

[사내 ERP]
   │  ① 내부 팀장이 필요한 자료를 수기 추출 (.xlsx / .csv)
   ▼
[사내 작업 PC]  ← 외부 컨설턴트가 원격 접속 (사내 자료는 PC 밖으로 반출 X)
   │  ② Claude Code가 데이터 → HTML 보고서 변환
   ▼
[Google Drive (법인 계정)]  ← 조직 내부자 한정 링크 권한
   │  ③ HTML 보고서 저장 및 링크 생성
   ▼
[대표이사]  ← 링크 클릭으로 조회 (PC / 모바일)

08 · Security

보안 체계 (핵심)

8-1. 데이터 처리 경로 (정확한 사실)

  • ERP 원본 파일(엑셀/CSV)은 사내 작업 PC에만 저장 — 외부 파일 공유·이메일 첨부 차단
  • 외부 컨설턴트는 원격 접속(RDP/VDI 등)으로 사내 PC 화면을 사용 (가능 시 클립보드·드라이브 매핑 차단)
  • AI(Claude Code) 분석 시점에는 분석에 필요한 데이터가 Anthropic 서버로 일시 전송됨 (TLS 1.2+ 암호화) — 자세한 정책은 8-5 참조
  • 민감 항목(개인정보·매출 상세·인사 정보 등)은 마스킹·집계 후에만 AI에 입력하는 원칙 적용
  • 외부 반출 가능한 산출물은 최종 HTML 보고서에 한정 (Google Drive 법인 계정 저장)

8-2. 보고서 접근 권한 통제

  • Google Drive 공유 정책: "링크가 있는 모든 사용자" 옵션 사용 금지
  • "조직 내부자에 한해 링크 접근" 또는 "지정 사용자만"으로 강제
  • 외부 공유가 불가피한 경우 만료일·다운로드 금지 설정
  • 감사 로그(누가·언제·어떤 파일을 열람) 표준 보존

8-3. 자격증명 통합 관리

  • 모든 법인 SaaS 계정의 ID/PW는 1Password 그룹 금고(Vault)에 보관
  • 개인 메모·메신저·엑셀로 비밀번호 공유 전면 금지
  • 외부 컨설턴트는 별도 그룹으로 분리 권한 부여
  • 계약 종료 시 그룹 탈퇴 1회로 모든 SaaS 접근 동시 차단

8-4. 계정 Lifecycle

  • 외부 인력 계정은 가입 시점부터 종료일을 명시
  • 분기별 계정·권한 점검 (미사용·역할 변동 정리)

8-5. AI(Claude Code) 데이터 정책 — Anthropic 공식

학습 사용모델 학습에 사용되지 않음 (Max 플랜은 기본 옵트아웃, Team/Enterprise는 약관상 미사용)

운영 원칙민감 항목 마스킹·집계 → AI 입력 → 결과 검수 → 보고서화
※ 출처: Anthropic 공식 데이터 사용 정책 (code.claude.com/docs). PoC 결과를 바탕으로 향후 더 강한 정책(ZDR / Enterprise) 전환 여부를 별도 검토.

09 · Decision Records

도구 채택 근거 (ADR)

ADR-001 · 분석·보고서 생성 엔진으로 Claude Code 채택

Context
  • 현재 정기 보고에 사용하는 ERP 데이터에 다각도 분석·요약·시각화 추가로 대표이사 의사결정 지원
  • 코드 작성·실행·파일 입출력·문서화를 한 환경에서 처리하여 구축·유지 비용 최소화
  • PoC 단계에서 빠른 가설 검증·결과물 반복 가능해야 함
  • 외부 컨설턴트가 떠난 뒤에도 회사가 산출물을 그대로 운용할 수 있어야 함
Decision

분석·보고서 생성 엔진으로 Anthropic Claude Code를 채택하며, 플랜은 Claude Max 5x 2계정(관리자 + 외부 컨설턴트)을 권장한다.

Consequences
  • 긍정: 코드+실행+문서화가 한 도구에서 가능, 산출물(코드·문서)이 회사 자산으로 잔존
  • 긍정: Max 플랜은 학습에 사용되지 않으며(옵트아웃 기본), 데이터 30일 후 자동 삭제
  • 긍정: 관리자도 동일 도구를 사용하므로 컨설턴트 산출물 검증·계약 종료 후 자체 운용 가능
  • 유의: AI 분석 단계에서 입력 데이터가 Anthropic 서버로 전송됨 → 마스킹·집계 원칙 적용 (8-1, 8-5)
  • 유의: 즉시 삭제(ZDR)는 Enterprise에서만 가능 → PoC 단계 미적용, 확대 시 재평가

ADR-002 · Google Workspace 법인계정 채택

Context
  • 5인 내외 소규모이지만, HTML 보고서를 외부 인사 포함 통제된 방식으로 공유해야 함
  • 개인 Gmail·드라이브 사용 시 자산 회수·감사 불가능
Decision

Google Workspace Business Plus 법인계정(5인)으로 도메인·메일·드라이브·권한 체계를 구축한다.

Plus 선택 이유: Vault(감사 로그·이디스커버리·보존정책)가 Plus부터 제공됨. 외부 인력이 접근하는 환경에서 누가 언제 어떤 파일을 열람·수정했는지 추적 가능해야 함.
Consequences
  • 긍정: 법인 도메인 기반 계정 → 개인 계정과 데이터 분리, 자산 회수 가능
  • 긍정: 조직 단위 공유 정책(외부 공유 차단·만료일) 표준 제공
  • 긍정: Vault 감사 로그 표준 제공 → 외부 인력 접근 이력 추적
  • 긍정: 사용자당 5TB 스토리지 → 보고서·자료 보관 여유
  • 부담: 인당 $22/월 (5인 약 $110/월) — Standard 대비 약 $40/월 더 비싸나 감사 기능 확보 가치 우선

ADR-003 · 1Password 법인계정 채택

Context
  • 향후 추가될 SaaS(분석·차트·번역 등) 자격증명을 안전하게 공유·회수해야 함
  • 외부 컨설턴트가 일부 자격증명을 사용해야 하나, 계약 종료 시 일괄 회수 필수
Decision

1Password Teams Starter Pack (10인 정액 $19.95/월) 법인계정 + 그룹 금고(Vault) 운영을 채택한다.

플랜 선택 근거: 본 안 인원은 5인 이내이므로 인당 단가 $7.99인 Business($39.95/월)보다 10인 정액 $19.95인 Starter Pack이 약 절반 가격. SSO·고급 감사 기능이 필요해지는 시점에 Business로 업그레이드.
Consequences
  • 긍정: 그룹 단위 권한 부여·회수 → 계약 종료 시 즉시 모든 비밀번호 접근 차단
  • 긍정: 비밀번호 사용·열람 감사 로그 보존
  • 긍정: 향후 추가 SaaS도 동일 체계로 흡수 가능 (확장성)
  • 부담: 월 $19.95 정액 → 5인 규모에서 비용 부담 가장 낮음
  • 한계: Starter Pack은 SSO·고급 감사 기능 미포함 → 인원 확대·SSO 필요 시 Business로 전환

리스크영향도대응 방안
외부 컨설턴트 원격 접속 중 사내 자료 유출 高 원격 접속 로그 보존, 파일 외부 전송·클립보드 차단 설정, 작업 시간대·범위 사전 합의
AI 분석 과정에서 민감 데이터 외부 송신 中 마스킹·집계 후 AI 입력 원칙, 개인정보·매출 상세는 사내 단계에서 분리. Max 플랜은 학습 미사용·30일 후 삭제 (Anthropic 공식)
Claude Code의 원본 파일 자동 읽기 (예: .env) 中 작업 PC에 분석용 폴더 분리, 자격증명·민감 파일은 별도 위치 보관, 권한 설정으로 접근 제한
컨설턴트 계약 종료 후 시스템 유지 불가 中 시나리오 B 채택 시 관리자가 동일 도구 사용 → 자체 운용 가능. 코드·문서 일체 회사 보유, 1Password로 자격증명 일괄 회수, 인수인계 문서 의무화
SaaS 라이선스 비용 누적 低 분기별 사용자 점검, 미사용 계정 즉시 비활성화
도구 장애·서비스 중단 低 원본 데이터는 사내 ERP에 보존 → 일시 중단 시에도 현행 수기 보고로 fallback 가능
환율 변동에 따른 비용 증가 低 USD 청구이므로 월별 KRW 비용 변동 가능 → 분기별 결산 시 점검
